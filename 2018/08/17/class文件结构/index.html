代码分析

先写一个简单的java类

KepTest.java

public class KepTest {
    private static final String a = "static final string";
    private static String b = "static string";
    private final String c = "final string";
    private String d = "normal string";

    public static void main(String[] args){
        System.out.println("Hello word");
    }

    public void test(){
        int i = 1;
        i = i + 1;
        System.out.println(i);
    }
}

KepTest.class

用javac对KepTest.java进行编译之后得到KepTest.class文件，我们随便用个文本编辑器将KepTest.class打开


因为class文件本质是以字节为单位的二进制流，这里打开的方式是按8位16进制展示，然后数据存放位置是严格按照上面结构图中的顺序来存储的，所以接下来我们就按照顺序来依次分析一波

开始分析

magic 魔数

class文件开头的4个字节被称之为”magic number”,唯一的作用为区分文件类型，表明这是一个能被jvm识别的class文件
由上面的图可以看到，这个魔数的值为0XCAFEBABE。

minor_version 次版本号 & major_version 主版本号

紧跟魔数之后的4个字节分别是2个字节的次版本号和2个字节的主版本号，版本号的作用是判断这个class文件能够被哪个版本的JDK执行，向下兼容
java的初始版本是45，JDK版本每做一次大版本升级，版本号就加一，比如JDK1.7能处理的class文件的版本号为小于 45+6 = 51的所有版本
同时由我们上面的图可以看到，主版本号是0X0033 换算成10进制就是51，说明这个class文件是由1.7版本的jdk编译的。

constant_pool_count 常量池大小

版本号之后是常量池，常量池前先使用了2个字节来存储常量池的大小，我们这里的常量池大小为0X003d,转成十进制为61，但是由于常量池大小计数是由1开始的，因此常量池大小为60

常量池

常量池中主要存放两大类常量：字面量和符号引用
字面量：文本字符串、final常量等
符号引用:

  1. 类和接口的全限定名
  2. 字段的名称和描述符
  3. 方法的名称和描述符
    摘自《深入理解java虚拟机》

常量池中的每一个常量都是一张表或者说是一个结构体,不同类型的数据结构也不同,但是相同的一点是所有常量的第一位都是一个字节(u1)的标志位(tag),用来表明当前常量的类型。
