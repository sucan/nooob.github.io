JVM GC

发表于 2018-06-02 | 分类于 JVM | 阅读次数

由于笔者只看过JVM的GC机制，因此本文中谈及的GC相关内容只针对JVM

什么是GC

gc(Garbage Collection)，中文名垃圾回收
个人的理解就是在程序执行期间对程序所占用的内存空间进行自动回收，使得开发者在写代码时不用关心内存的分配与回收

如何进行GC

其实如何进行GC就和怎样把大象放进冰箱里一样，就两步:

1. 把冰箱门打开
2. 把大象放进去

同样的GC也就两步：

1. 找到需要被回收的对象
2. 回收掉，释放对应内存空间

当然实际操作肯定没有我说得这么简单，但是总体的思想也就是这样，下面会继续介绍一下具体细节

如何判断对象是否需要被回收

很容易就能想到的是当一个对象不再被引用的时候，那么这个对象就是可以被回收的对象

JVM 所采用的判断对象是否引用的算法为可达性分析算法

简单的说，就是选取一些特定的对象作为GC roots节点,然后对这些节点做BFS，最终得到的对象集合就是有引用的对象，不在这个集合中的就是需要被回收的对象

其中，GC roots 包括：

1. 虚拟机栈中引用的对象
2. 方法区中类静态属性引用的变量
3. 方法区中常量引用的对象
4. native方法引用的对象

GC算法

我们现在知道了哪些对象需要被回收之后就要开始进行对象的回收啦，那么对象又该如何进行回收呢

JVM 所使用的GC算法(或者叫GC策略一共有三种)：

标记-清除算法(mark-Sweep)

这是最基础，最容易想到的方式了，实现方式也很简单，就是标记出需要回收的对象，然后直接将该对象所占用的内存空间释放掉

虽然看似很合理，也确定达到了回收空余内存空间的效果，但是由于被回收的对象可能分散于不同的内存地址中，所以这种方式的缺点在于

可能会产生大量的内存碎片，在进行大对象的内存分配时会因为找不到足够大的连续内存空间而再次触发GC，虽然此时可使用的内存空间远远大于需要分配的空间
